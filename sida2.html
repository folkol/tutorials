Mer om Objects

Ett JavaScript Object är som sagt en samling properties. En property är ett namn associerat med ett värde.

Det finns lite övriga sorters properties som inte är relevanta just nu, men är man intresserad så kan man slänga ett öga på specen.

Det finns som sagt ett antal olika sätt att skapa JavaScript Objects, alla med sina för- och nackdelar.

Skapa några Object:
var baz = {};
var bar = Object.create(null);
var foo = new Object();
Att tilldela objekten properties:
var foo = new Object();
foo.bar = 2;
foo.baz = "Hello";
foo.qux = {};
Att skapa objekt med properties:
var foo = { bar: "Hello" }; // foo.bar = "Hello";

Syntaxen med måsvingarna kallas "Object literal syntax", och är helt enkelt objektet beskrivet i JSON-format, med undantaget att namnet på de properties man skapar inte behöver vara inom citattecken.

Kort sagt så är Object literal syntax en startmåsvinge, sedan ett antal kommaseparerade properties på formen namn:value, och sedan en avslutande måsvinge.

Object.create tar ett argument, som vi i exemplet ovan sätter till null. Vi kommer att gå igenom vad det är för lustig parameter i en senare övning.

Man måste inte deklarera properties i förväg, de kan skapas på godtyckligt Object genom att bara tilldela ett värde till en property med ett nytt namn. Detta gäller även diverse Objects som JavaScript-runtimen har gjort tillgängliga för oss att kommunicera med.

Det finns två sätt att läsa / skriva properties, antingen genom den syntax vi såg ovan, eller genom square bracket notation. Lägg märke till att namnet inom brackets är en string.

Properties, både med dot notation och square bracket notation:
var foo = new Object();
foo.bar = 2;
foo['baz'] = 'Hello';

Om man vet namnet på den property som man vill skriva eller läsa så är det ingen skillnad på dessa två syntaxer, men om man har propertynamnet i en variabel så måste man använda sig av sen senare varianten.
