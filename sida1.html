Variabler och typer

En variabel i JavaScript deklareras med ett Variable Statement, vilket består av ordet 'var' följt av ett namn.

Ett statement som deklarerar en variabel vid namn 'foo':
var foo;

En variabel i JavaScript är otypad, men det värde som variabeln för tillfället refererar till är typat och har en av följande typer:

  • Boolean
  • Null
  • Undefined
  • Number
  • String
  • Object

Man kan få en textsträng som representerar en variabels typ genom att använda operatorn "typeof". Se specen för mer detaljer.

Vi ska gå igenom typerna lite hastigt och kommer i senare slides gå in lite mer på djupet av typen Object

Boolean

Kan anta värdena true och false. Här passar vi även på att visa kommentars-syntaxen.

Att tilldela variabler boolean values:
var foo = true;
var bar = false;
var fooType = typeof foo; // 'boolean'

Null

Kan endast anta värdet null.

Att sätta en variabeln till null:
var foo = null;

Undefined

Kan endast anta värdet undefined. Alla variabler har värdet undefined innan de tilldelas något annat värde. Det finns fler situationer där man träffar på undefined, men vi kommer till det senare.

Man tilldelar generellt inte värdet undefined till en variabel.

Variabler är undefined tills man tilldelat dem ett värde:
var foo; // undefined

Number

Ett 64 bits tal, enligt IEEE 754. D.v.s. ett gäng rationella tal, samt NaN, -Inf och +Inf.

Numbers är vad det låter som, ett aritmetiskt värde. Man kan konvertera en textsträng till ett Number genom toNumber (Eller toInteger, om man bara vill ha heltal).

Number values:
var foo = 2;
var bar = 3.14;
var baz = parseInt('1337');

String

En serie av UTF-16 code points. Används oftast för text-data.

Man kan läsa ut strängens längd m.h.a. propertien ".length". Vi återkommer till vad properties är under Object.

Man skapar textsträngar genom antingen enkla eller dubbla citattecken. Båda fungerar lika bra, och bägge typer är ungefär lika vanliga. Jag använder enkla om inte texten jag ska lagra innehåller enkla citattecken i sig.

Man kan konkatenera strängar med String.concat eller +.

Skapa textsträngar:
var foo = 'Hello';
var bar = "Hello there";
var baz = "Hello".concat(" there");
var qux = "Hello" + " there";

Object

Ett JavaScript Object är en samling properties. En property är ett namn associerat med ett värde, värdet kan vara något av de primitiva ovan eller i sin tur ett Object.

Det finns lite övriga sorters properties som inte är relevanta just nu, men är man intresserad så kan man slänga ett öga på specen.

Det finns ett antal olika sätt att skapa JavaScript Objects, alla med sina för- och nackdelar.

Skapa några Object:
var foo = new Object();
var bar = Object.create(null);
var baz = {};
Att ge objekten properties:
var foo = new Object();
foo.bar = 2;
foo.baz = "Hello";
foo.qux = new Object();
Att skapa object med properties:
var foo = { bar: "Hello" }; // foo.bar = "Hello";

Syntaxen med måsvingarna kallas "Object literal syntax", och är helt enkelt objektet beskriver i JSON-format, med undantaget att namnet på de properties man skapar inte behöver vara inom citattecken - då det inte skulle kunna vara något annat än propertnamn man menar i den kontexten.

Kort sagt så är Object literal syntax en start måsvinge, sedan ett antal kommaseparerade properties på formen namne:value.

Object.create tar en parameter, som vi i exemplet ovan sätter till null. Vi kommer att gå igenom vad det är för lustig parameter i nästa övning.
