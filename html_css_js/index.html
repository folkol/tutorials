Web page tutorial

  • HTML – Hypertext Markup Language
  • CSS – Cascading Style Sheets
  • JS – JavaScript

In this tutorial, we are going to create a web page – this web page!

The tutorial consists of three parts, one for each of the basic web page building blocks: HTML, CSS and JS.

  • HTML (Hypertext Markup Language): A (markup-) language for structuring the contents of a web page. Declares what various parts of the web page are.
  • CSS (Cascading Style Sheets): A set of style rules that define how the various elements of the web page should look.
  • JS (JavaScript): A programming language. JavaScript can be used to dynamically update the page, e.g. manipulate the DOM or otherwise instruct the Browser to do something.

Make sure that you are using a plain text editor (Notepad, Sublime Text), and not a 'rich text' word processor that embeds additional information in the file, when working through this tutorial.

HTML – Hypertext Markup Language

In this section, we will introduce HTML, elements, attributes and the DOM.

HTML is a markup language that adds meaning and structure to a text by the means of tags. HTML tags defines what parts of the document ARE.

Getting started

  1. Create a folder for your web page.
  2. Create an empty text file with the file extension '.html' (e.g. 'index.html').
  3. Open the file with a web Browser, it should look like a blank page.
  4. Copy all text from this page and paste it into your html file. Refresh the Browser – it should display a blob of text.

Markup

The reason why your web page looks like poop is that it has got no encoded structure. We can begin adding some structure by tagging – or marking up – the headline as such.

Enclosing a section of the text in tags like that will declare that part of the page to be a heading.

There are six levels of headings, h1 means "the most important heading", h2 means "the second most important heading" etc.

Elements

A section of the document tagged up like this is called an "element". There are a few different kinds of elements, for example "normal elements", "raw text elements" and "void elements".

When a "User agent" (e.g. a Browser) is parsing (i.e. reading through and interpreting) HTML code, it will build up a tree structure corresponding to these elements in memory. This tree is called 'the DOM' (Document Object Model), and the nodes in this tree are called "DOM Nodes".

The DOM is the Browsers in-memory representation of the web page.

Inspect your the heading with your Browser's Developer Tools. You will notice that it is not alone, but sits in a hierarchy of parent nodes.

The reason for this is that HTML has got a notion of optional tags. Some elements are mandatory and will be created according to some scheme whether you actually add their tags to your HTML code or not. My suggestion is that you add the tags for the root element (html), the document metadata element (head) and the document content element (body) to your HTML code.

More headings!

Mark the rest of the heading elements with suitable h-tags (h2, h3, h4 etc.).

Lists

There are two kinds of lists in HTML. Ordered - or numbered – lists, and unordered – or bulleted – lists.

An ordered list is declared with the "ol"-tag, and its list items are declared with the "li"-tag.

An unordered list is declared with the "ul"-tag, and its list items are also declared with the "li"-tag.

Images

Images can be embedded in the text by the "img"-tag. The "src" attribute is a URL to the image. This can be absolute (i.e. contain domain and path) or relative (i.e. only contain a path relative to the current html document.

  1. Go ahead and add the images to your web page.

HTML Entities

Summary

  • HTML pages are text files annotated, or marked up with, tags.
  • HTML pages are parsed by user agents, for example web browsers, into a DOM tree.
  • The primary purpose of HTML is to describe the structure and the semantics of the various parts of the page.

CSS – Cascading Style Sheets

HTML defines what various parts of the web page are, CSS defines how they should look.

In this section, we will introduce CSS, selectors, declaration blocks and properties.

The Browser has got a few default style rules for various elements, that's why headings look different than body text and so on (The Browser Developer Tools can list what style rules that are applied do a certain element – both default rules as well as rules applied by web page CSS).

Getting started

CSS can be written in its own file, and linked to the web page with a LINK tag – or it can be written in the HTML source itself – either as an attribute of the element tag or in its own STYLE tag.

Keeping the CSS code in its own file is best practice – so that is what we will do.

  1. Create a new, empty, text file called "style.css" (the name is not significant, but it is a common name for the main style sheet).
  2. Link this style sheet to the web page by adding a 'LINK' tag as a child to the to the 'HEAD' element.

CSS rules

CSS code consists of a set of styles rules. Each rule consists of a selector and a block of style properties.

There are a lot of selectors that can be used. We will go through some of the basic ones, but in the general case, the selector declares what elements that will be targeted by the corresponding block of style properties.

In this section, we will introduce JS, Script-tags, functions and event listeners.

Getting started

JavaScript can be used to program the web page, both when the page is loaded – and in response to events that happens on the page.

The gist of it is that the web page isn't just a dead image – but a living contraption!. There are a lot of things going on, even on a seemingly static page. Events are triggered when the user scrolls the page, then the mouse moves across the page and whatnot. These things trigger "events", and the programmer can register "Event listeners" that will be triggered by these events.
